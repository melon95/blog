instanceof

instanceof 用于检测构造函数的 prototype 属性是否出现在某个实例对象的原型链上。


[] instanceof Array // true
[] instanceof Object // true

new Object() instanceof Object // true
// 加()是因为{}会被解析成代码块
({}) instanceof Object // true

// 这些构造函数都是函数，函数是特殊的对象
Number instanceof Function // true
String instanceof Function // true
Array instanceof Function // true
Object instanceof Function // true
Function instanceof Object // true
Object instanceof Object // true



// 基本类型
'' instanceof String // false
// new String()是返回一个String对象
new String() instanceof String // true

1 instanceof Number // false
// new Number()是返回一个Number对象
new Number() instanceof Number // true

true instanceof Boolean // false
new Boolean() instanceof Boolean // false

// 下面两种都不能使用new操作符
BigInt(2) instanceof BigInt // false
Symbol('') instanceof Symbol // false

手动实现简单 instanceof ：

  function Instanceof (leftValue, rightValue) {
    let prototype = leftValue.__proto__
    while (prototype) {
      if (prototype === rightValue.prototype) {
        return true
      } else {
        prototype = prototype.__proto__
      }
    }
    return false
  }

  • Array ：跟 Object 是一样的情况。下面是对 Array 的一个总结：
    • 如果数组的长度为0，则转换结果为0。空数组转换成基本数据类型就是 '' ，在转换成 Number 类型就是0。
    • 如果数组的长度为1，则转换结果就相当于直接对数组的第一项直接调用 Number 函数
    • 如果数组的长度大于1，则转换结果为NaN。数组长度大于1，转换成基本数据类型就是有逗号拼接的字符串，转换成 Number 就是 NaN 。

相等比较

JavaScript 中相等比较分为两种：

  1. == ：允许在相等比较过程中进行强制类型转换
  2. === ：不允许在相等比较过程中转换类型

=== 完全像我们预期的那样执行，但是 == 有可能会出乎于我们的意料之外。
下面分析一下 == 的一些常用情况：

  • 字符串和数字转换：会尝试把字符串转换为数字在进行比较
  • 其他类型和 Boolean 转换：先把 Boolean 转换为 Number 在进行比较操作
  • 引用类型数据和基本属性数据比较：把引用类型转换为原始值(Primitive)，再进行比较



[] == 0 // true
[] == false // true
// [] == 0 ![] == false false == 0
[] == ![] // true

[] == 0 // true [].toString() == ''
[1] == 1 // true [1].toString() === '1'
[2] == 2 // true [2].toString() === '2'
[1,2] == '1,2' // true [1,2].toString() === '1,2'



a == 1 && a == 2
