代码的生命周期

一段 JavaScript 代码在运行前大体会经历3个阶段（统称为编译）：

  • 词法分析：将代码分解成（对编程语言来说）有意义的代码块，这些代码块被称为词法单元块（token）
  • 语法分析：将词法单元流转换成 AST 树
  • 代码生成：将 AST 树转换为可执行的代码

编译完成后，开始执行代码。首先创建一个执行环境，也就是执行上下文，执行上下文包括了代码在执行过程中所有会用到的变量。
执行完成后就会销毁这个执行上下文，代码的生命周期也就结束了。要弄懂作用域链、变量如何进行查找的一些细节，就先要弄清楚执行上下文。

执行上下文

执行上下文在本质上都是一种作用域层。每个代码段开始执行的时候都会创建一个新的上下文来运行它，并且在代码退出的时候销毁掉。
当一段 JavaScript 代码在运行的时候，它实际上是运行在执行上下文中。下面3种类型的代码会创建一个新的执行上下文：

  • 全局上下文是为运行代码主体而创建的执行上下文，也就是说它是为那些存在于JavaScript 函数之外的任何代码而创建的。
  • 每个函数会在执行的时候创建自己的执行上下文
  • eval 函数也会创建一个新的执行上下文

执行上下文中包括了变量、this、外部环境（作用域链）

  • 变量环境（Lexical Environment）：保存函数声明、var 声明的变量
  • 词法环境（VariableEnvironment）：保存let、const、class等定义的变量
  • outer：指向上层作用域（全局作用域没有上层作用域）
  • this：执行过程 this 的值 :::info 词法环境是为了兼容块级作用域 ::: 如果之前没有了解过执行上下文，可能会比较懵，下面就借用一段代码来稍微解释下：

作用域

作用域是指在程序中定义变量的区域，该位置决定了变量的生命周期。通俗地理解，作用域就是变量与函数的可访问范围，即作用域控制着变量和函数的可见性和生命周期。
在 JavaScript 中有4种作用域：

  • 全局作用域中的对象在代码中的任何地方都能访问，其生命周期伴随着页面的生命周期。
  • 函数作用域就是在函数内部定义的变量或者函数，并且定义的变量或者函数只能在函数内部被访问。函数执行结束之后，函数内部定义的变量会被销毁。d
  • 块级作用域

在每个执行上下文中的变量环境中，都包含来一个外部引用，用来指向外部的执行上下文，当一个变量在当前的执行上下文中的词法环境和变量环境没找到时，就回去外部执行上下文中去找。把这个一级一级查找的链条就称为作用域链。
当理解了上面的这些概念，相信你就能很轻松的理解闭包了。

变量提升

变量提升，是指在JavaScript代码执行过程中，JavaScript 引擎把变量的声明部分和函数的声明部分提升到代码开头的“行为”。变量被提升后，会给变量设置默认值，这个默认值就是我们熟悉的undefined。

在 ES6 之前，不管是变量还是函数都会被提升到作用域的顶层被初始化。这是因为代码在编译阶段会被保存到变量环境中：

  • 函数声明：首先提取函数（优先级高）
  • 变量声明：提取变量并初始化值为 undefined（优先级低）

变量提升会带来一些问题：

  • 变量容易在不知情的情况下被覆盖
  • 应该销毁的变量没有被销毁（例如：for 循环中的 var i）

但是在 ES6 后，JavaScript 出现了 let、const、class，在很大程度上改善了之前那种让人迷惑的代码。
用 let、const、class声明的变量，在编译阶段也会被处理，但是不会被初始化（no-initialize），并且也不能在变量声明位置之前访变量（暂时性死区）。

WARNING

块级作用域不会影响 var，但会影响函数声明

console.log(test) // undefined
console.log(haha) // undefined
{
  console.log(haha) // haha() {}
  var test = '1'
  function haha() {}
}
console.log(test) // 1
console.log(haha) // haha() {}

变量查询

变量查询分为两种：

  • 获取变量的值（RHS）
  • 获取变量本身（LHS）

这样可能有点难以理解，下面用几个例子来简单阐述下：

let a = 2 // LHS查询，不在乎变量的值，只需要获取变量本身，然后把2赋值给a
let b = a // 对b来说也是LHS查询，但是对a来说是RHS查询，只需要获取a的值，然后赋值给b

LHS 和 RHS除了概念上的不同，当对一个为定义的变量分别采取 LHS 和RHS 的时候，也会有不同的结果：

a = 2 // 对一个未声明的变量采取LHS查询，会自动声明一个变量（严格模式除外）
console.log(b) // 对一个未声明的变量采取RHS查询：ReferenceError: b is not defined

总结

一段代码在执行前会先经历编译阶段，编译阶段会发生变量提升。在代码执行时，先创建一个执行上下文，执行上下文中包含了：变量环境、词法环境、outer(外部环境指针)、this。
JavaScript 中有3种作用域，分别是：全局作用域、函数作用域、块级作用域。作用域中有两种方式进行变量查询：LHS 和 RHS。
