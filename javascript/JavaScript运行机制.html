JavaScript运行机制

介绍

JavaScript 是一种单线程的语言，同一时间只能处理一个任务。必须等执行完上一个任务，才会执行下一个任务。
这样的设计在某些情况下，会带来很大的不便。于是就出现了同步任务和异步任务。

如果JS是多线程，在同一时间有两个线程，都操作同一个DOM就会出线问题。

同步任务

在 JavaScript 主线程中按顺序执行，会生成一个执行栈。

异步任务

在主线程外，还存在一个任务队列(task queue)。当异步任务执行完后，会把它加入到任务队列中，当执行栈里面的同步任务全部执行完的时候，就把任务队列中的任务去回到主线程中执行，然后循环这个过程。

主线程从任务队列中读取任务,这个过程时循环不断的,这种运行机制也称为事件循环(Event Loop) .

任务队列是一个先进先出的数据接口,排在前面的优先的主线程取回执行


微任务(microtask)和宏任务(macrotask):
异步任务又细分为微任务和宏任务.

  • 微任务
    - setTimeout
    - setInterval
    - setImmediate
    - MutationObserver

- ...

  • 宏任务
    - Promise.then()
    - Promise.catch()
    - promise.finally()
    - process.nextTick- script代码块
    - ...
