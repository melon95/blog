原型链和实现继承的方式

介绍

当谈到继承时，JavaScript 只有一种结构：对象。每个实例对象（ object ）都有一个私有属性（称之为 proto ）指向它的构造函数的原型对象（prototype ）。该原型对象也有一个自己的原型对象( proto ) ，层层向上直到一个对象的原型对象为 null。根据定义，null 没有原型，并作为这个原型链中的最后一个环节。
JavaScript 对象是动态的属性“包”（指其自己的属性）。JavaScript 对象有一个指向一个原型对象的链。当试图访问一个对象的属性时，它不仅仅在该对象上搜寻，还会搜寻该对象的原型，以及该对象的原型的原型，依次层层向上搜索，直到找到一个名字匹配的属性或到达原型链的末尾。

被构造函数创建的实例对象的 [[Prototype]] 指向构造函数的的 prototype 属性。

原型链继承

function Prototype(msg) {
  this.msg = msg
}
Prototype.prototype.outputMsg = function() {
  console.log(this.msg)
}

function Instance() {

}
// 这行代码不能写Instance函数内部，这里也不用纠正Instance.prototype.constructor的指向
// 因为Instance.prototype没有这个属性
// 当执行new Prototype()的时候没有办法灵活的传递参数
Instance.prototype = new Prototype()

const instance = new Instance()
console.log(instance)

instance instanceof Instance // true
instance instanceof Prototype // true

缺点：

  • 没办法灵活的传递参数
  • 多个实例共同引用同一个原型

借用构造函数+原型链继承

function Prototype(msg) {
  this.msg = msg
}
Prototype.prototype.outputMsg = function() {
  console.log(this.msg)
}

function Instance(msg) {
  // 让Instance实例实现Prototype上面的实例方法
  Prototype.call(this, msg)
}
// 让Instance继承Prototype.prototype上面的原型方法和属性
Instance.prototype = new Prototype()

const instance = new Instance('instance')
console.log(instance)

instance instanceof Instance // true
instance instanceof Prototype // true

借用构造函数+Object.create()

function Prototype(msg) {
  this.msg = msg
}
Prototype.prototype.outputMsg = function() {
  console.log(this.msg)
}

function Instance(msg) {
  Prototype.call(this, msg)
}、
// 这里不能直接令Instance.prototype = Prototype.prototype
// 如果这样做的话往Instance.prototype上添加属性，也会改变Prototype.prototype
Instance.prototype = Object.create(Prototype.prototype)

const instance = new Instance('instance')
console.log(instance)

instance instanceof Instance // true
instance instanceof Prototype // true

Class继承

class Prototype {
  constructor(msg) {
    this.msg = msg
  }
  output() {
    console.log(this.msg)
  }
}
class Instance extends Prototype {
  constructor(msg) {
    super(msg)
  }
}
const instance = new Instance('instance')
console.log(instance)


instance instanceof Instance // true
instance instanceof Prototype // true

总结

虽然上面罗列了四种方法，但是在实际使用中还是推荐使用第四种或者第三种，前面两种只是作为一个介绍的过度。
