## 作用域
作用域的工作方式有两种占统治地位的模型。其中的第一种是最最常见，在绝大多数的编程语言中被使用的。它称为 **词法作用域**，我们将深入检视它。另一种仍然被一些语言（比如 Bash 脚本，Perl 中的一些模式，等等）使用的模型，称为 **动态作用域**。JavaScript使用的就是词法作用域。<br />先来看一段代码<br />

```javascript
  var a = 'window'
  function foo() {
    console.log(a)
  }
  function bar() {
    var a = 'bar'
    foo()
  }
  bar() // window
```


```javascript
  var a = 'window'
  function foo() {
    var a = 'foo'
    console.log(a)
  }
  function bar() {
    var a = 'bar'
    foo()
  }
  bar() // foo
```


```javascript
  var a = 'window'
  function bar() {
    var a = 'bar'
    foo()
    
    function foo() {
      console.log(a)
    }
  }
  bar() // bar
```

<br />词法作用域是由变量或者函数声明的位置决定，是静态的，一般来说都不会变动。当在当前作用域找不到对应的变量或者方法时，会尝试去包含当前作用域的父级作用域去寻找，直到找到全局作用域或者找到第一个匹配时就会停止。作用域一层一层的查找就表示了一条**作用域链**。

## 闭包
闭包就是指**有权访问另一个函数作用域中的变量的函数**。<br />比如：

```javascript
  function foo() {
    var a = 'foo'
    function bar() {
      console.log(a)
    }
    return bar
  }
  var a = 'window'
  var baz = foo()
  baz() // foo
```

<br />函数 `bar()` 对于 `foo()` 内的作用域拥有词法作用域访问权。但是之后，我们拿起 `bar()` ，这个函数本身，将它像值一样传递。在这个例子中，我们 `return bar` 引用的函数对象本身。<br />在执行 `foo()` 之后，我们将它返回的值（我们的内部 `bar()` 函数）赋予一个称为 `baz` 的变量，然后我们实际地调用 `baz()` ，这将理所当然地调用我们内部的函数 `bar()` ，只不过是通过一个不同的标识符引用。<br />`bar()` 被执行了，必然的。但是在这个例子中，它是在它被声明的词法作用域 外部 被执行的。<br />`foo()` 被执行之后，一般说来我们会期望 `foo()` 的整个内部作用域都将消失，因为我们知道 引擎 启用了 垃圾回收器 在内存不再被使用时来回收它们。因为很显然 `foo()` 的内容不再被使用了，所以看起来它们很自然地应该被认为是 消失了。<br />但是闭包的“魔法”不会让这发生。内部的作用域实际上 依然 “在使用”，因此将不会消失。谁在使用它？函数 `bar()` 本身。<br />有赖于它被声明的位置， `bar()` 拥有一个词法作用域闭包覆盖着 `foo()` 的内部作用域，闭包为了能使 `bar()` 在以后任意的时刻可以引用这个作用域而保持它的存在。<br />`bar()` 依然拥有对那个作用域的引用，而这个引用称为**闭包**。<br />**闭包就是当一个函数即使是在它的词法作用域之外被调用时，也可以记住并访问它的词法作用域。**<br />
<a name="N244C"></a>
#### 优点

- 私有化变量
- 保持对词法作用域的访问能力
<a name="Sm9SE"></a>
#### 缺点

- 内存使用量变大
