# 跨域解决方案
<a name="2jXub"></a>
## 介绍
由于浏览器同源策略的存在，请求资源的时候，如果双方不是同源既协议、域名、端口至少有一个不同，就会发生跨域。
<a name="1FdnH"></a>
## CORS
跨域资源共享(CORS) 是一种机制，它使用额外的 HTTP 头来告诉浏览器  让运行在一个 origin (domain) 上的Web应用被准许访问来自不同源服务器上的指定的资源。当一个资源从与该资源本身所在的服务器**不同的域、协议或端口**请求一个资源时，资源会发起一个**跨域 HTTP 请求**。<br />浏览器将 CORS 请求分为两种：**简单请求**和**非简单请求**。<br />**简单请求**必须同时满足下面两个条件：<br />（1) 请求方法是以下三种方法之一：

- HEAD<br />
- GET<br />
- POST1<br />

（2）HTTP的头信息不超出以下几种字段：

- Accept<br />
- Accept-Language<br />
- Content-Language<br />
- Last-Event-ID<br />
- Content-Type：只限于三个值`application/x-www-form-urlencoded`、`multipart/form-data`、`text/plain`<br />

非简单请求就是不是简单请求的其他所有请求。<br />浏览器对于这两种请求的处理方式是不同的。
<a name="iJ7E4"></a>
### 简单请求
对于简单请求浏览器直接发出 CORS 请求。

![](https://cdn.nlark.com/yuque/0/2020/png/299895/1595309834399-e870a269-80f5-4229-aa60-267e898705c6.png#align=left&display=inline&height=490&margin=%5Bobject%20Object%5D&originHeight=490&originWidth=1023&size=0&status=done&style=none&width=1023)<br />浏览器在请求头中添加：

1. Origin：指名当前请求的域名<br />

服务器在响应头中添加：

1. Access-Control-Allow-Origin：用来指定允许访问的域名，如果值为 `*` 则标识运行所有域名请求<br />
1. Access-Control-Expose-Headers：如果浏览器请求包括Access-Control-Request-Headers字段，则该字段是必需的。它也是一个逗号分隔的字符串，表明服务器支持的所有头信息字段<br />
1. Access-Control-Allow-Credentials：该字段可选。它的值是一个布尔值，表示是否允许发送Cookie。设为true，即表示服务器明确许可，Cookie可以包含在请求中，一起发给服务器。这个值也只能设为true，如果服务器不要浏览器发送Cookie，删除该字段即可。<br />
<a name="SV3C9"></a>
### 非简单请求
对于非简单请求，则会在正式发起请求之前，会发送一次预检请求(OPTIONS方法)，从而获知服务端是否允许该跨域请求，以及可以使用哪些HTTP动词和头信息字段。

![](https://cdn.nlark.com/yuque/0/2020/png/299895/1595309849613-50829132-f8ce-4406-9638-f8b032ff5cac.png#align=left&display=inline&height=553&margin=%5Bobject%20Object%5D&originHeight=553&originWidth=521&size=0&status=done&style=none&width=521)<br />浏览器在请求头中添加 ：

1. Access-Control-Request-Method: 请求方法<br />
1. Access-Control-Request-Headers: 请求携带的自定义头部字段，例如：X-PINGOTHER<br />

服务器会在响应头中添加：

1. Access-Control-Allow-Methods：该字段必需，它的值是逗号分隔的一个字符串，表明服务器支持的所有跨域请求的方法。<br />
1. Access-Control-Allow-Headers：如果浏览器请求包括Access-Control-Request-Headers字段，则该字段是必需的。它也是一个逗号分隔的字符串，表明服务器支持的所有头信息字段<br />
1. Access-Control-Allow-Credentials：该字段可选。它的值是一个布尔值，表示是否允许发送Cookie。设为true，即表示服务器明确许可，Cookie可以包含在请求中，一起发给服务器。这个值也只能设为true，如果服务器不要浏览器发送Cookie，删除该字段即可。<br />
1. Access-Control-Max-Age：该字段可选，用来指定本次预检请求的有效期，单位为秒,在有效期内不会再次发送预请求

## JSONP
**JSONP**（**JSON with Padding**）是数据格式JSON的一种“使用模式”，可以让网页从别的网域要数据。<br />由于同源策略，一般来说位于server1.example.com的网页无法与 server2.example.com的服务器沟通，而HTML的 `<script>`元素是一个例外。利用 `<script>`元素的这个开放策略，网页可以得到从其他来源动态产生的JSON数据，而这种使用模式就是所谓的 **JSONP**。<br />**JSONP **只支持 GET 方法。<br />可以直接通过 `script` 使用
```html
<script src="http://server2.example.com/RetrieveUser?UserId=1823&callback=callbackFunction">
```
也可以封装一下：
```javascript
function jsonp({url, callback, params}) {
    return new Promise((resolve, reject) => {
      let scripts = document.createElement('script')
      window[callback] = function (data) {
        resolve(data)
        document.removeChild(scripts)
      }
      const keys = Object.keys(params)
      let arr = []
      keys.foreach((key) => {
        arr.push(`${key}=${params[key]}`)
      })
      arr.push(`callback=${callback}`)
      scripts.src = `${url}?${arr.join('&')}`
      document.body.appendChild(scripts)
    })
  }
```

## PostMessage
**window.postMessage() **方法提供了一种受控机制来规避此限制，只要正确的使用，可以安全地实现跨源通信。<br />通过如下方式发送消息：
```javascript
otherWindow.postMessage(message, targetOrigin, [transfer]);
```

- otherWindow：其他窗口的一个引用
- message：将要发送到其他 window的数据
- targetOrigin：通过窗口的origin属性来指定哪些窗口能接收到消息事件，其值可以是字符串"*"（表示无限制）或者一个URI
- transfer：可选参数，是一串和message 同时传递的 `Transferable` 对象. 这些对象的所有权将被转移给消息的接收方，而发送一方将不再保有所有权。

通过监听 `message` 事件来接受消息：
```javascript
window.addEventListener("message", receiveMessage)
function receiveMessage(event) {
  const {
    data, // 从其他 window 中传递过来的对象。
    origin, // 调用 postMessage  时消息发送方窗口的 origin 
    source, // 对发送消息的窗口对象的引用
  } = event
}
```
